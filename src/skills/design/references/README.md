# Design References

Use these references during `/design` to keep outputs consistent, intentional, and implementation-ready.

## Files

- `research-playbook.md`: where to search, what to capture, and when to stop.
- `typography.md`: type hierarchy, spacing, and readability rules.
- `color-and-contrast.md`: color roles, palette strategy, and contrast checks.
- `motion-and-interaction.md`: motion timing, state transitions, and reduced-motion policy.
- `content-and-conversion.md`: headline logic, trust strategy, and CTA hierarchy.
- `implementation-qa.md`: build and QA checklist before handoff.
- `anti-slop.md`: patterns to avoid and fast heuristics for quality.

## How to use

1. Start with `research-playbook.md`.
2. Build visual direction using typography, color, and motion references.
3. Shape copy and persuasion with `content-and-conversion.md`.
4. Run `implementation-qa.md` and `anti-slop.md` before final output.
