# Implementation QA Checklist

Run this checklist before final handoff.

## Structure

- Semantic landmarks are present (`header`, `main`, `nav`, `footer` as needed)
- Heading hierarchy is valid and scannable
- Components are reusable and consistently named

## Visual System

- Type tokens, color tokens, and spacing tokens are defined
- Buttons, inputs, and cards share a clear family style
- Empty, loading, and error states are designed

## Interaction

- Keyboard navigation works end-to-end
- Focus states are visible and consistent
- Disabled and error states are explicit

## Responsive

- Mobile-first layout works at common breakpoints
- No clipped critical content
- Touch targets are usable

## Accessibility

- Color contrast meets baseline
- Inputs have labels and error guidance
- Motion respects reduced-motion preference

## Quality Gate

- Primary user job is obvious in 5 seconds
- Primary CTA is unambiguous
- Design rationale can be explained section by section
- Output feels specific to this product, not generic
