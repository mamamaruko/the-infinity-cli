# Content and Conversion Reference

## Purpose

Good UI copy removes hesitation and guides users toward the right decision.

## Messaging Stack

- Hook: one clear promise
- Support: what this solves now
- Proof: evidence, numbers, customer signal
- Objection handling: answer the top risk
- CTA: next step in plain language

## Headline Rules

- Prefer specific outcomes over abstract slogans
- Use verbs and concrete nouns
- Keep one primary message per section

## Trust Elements

Pick 2-4 depending on context:

- customer logos
- usage numbers
- security/compliance badges
- testimonial with role/company
- before/after example

## CTA Hierarchy

- One primary CTA per viewport cluster
- Secondary CTA must not visually compete
- Labels should describe the result (for example: "Start free trial")

## Objection Handling

Map likely concerns to direct responses:

- Price concern -> ROI, trial, guarantee
- Trust concern -> proof, policy, security
- Complexity concern -> setup time, migration support, templates

## Anti-Patterns

- Vague motivational copy
- Competing CTA styles in one section
- Long paragraphs where bullets are clearer
